# 简历优化建议

## 项目描述建议
- 用“做了什么 + 用了什么技术 + 达到了什么结果”的结构描述
- 不要只写技术名，要写业务场景和落地过程
- 项目经历优先突出与目标岗位相关的技术点

## AI 项目建议
- 写清楚是否接入大模型
- 写清楚是否做了 RAG / Agent / Tool Use / Memory
- 写清楚后端服务、接口设计、部署和测试内容
- 写清楚使用的模型、框架和工程化能力

## 后端项目建议
- 写清楚数据库设计
- 写清楚接口设计
- 写清楚权限控制
- 写清楚部署与测试
- 写清楚性能优化或异常处理

## 常见问题
- 项目描述太泛，缺少细节
- 只写“参与开发”，没有体现个人贡献
- 只列技术栈，没有说明系统流程
- 简历与岗位关键词匹配度不足
